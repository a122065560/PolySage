一、项目概述
开源的多AI协作工具。用户本地运行，无需任何API Key。通过 Playwright 控制已登录的 AI 网页，支持两种核心模式：

托管模式：2位AI全自动讨论，哨兵标记结束，支持灵活的拍板机制。
圆桌模式：用户主导，2-3位AI参与，手动控制发言节奏，实时引导讨论。

1. AI平台管理（配置与登录）
配置文件：所有配置存储在 config.json 中。结构如下：

  {
    "chrome": {
      "debug_port": 9222,
      "user_data_dir": "~/.roundtableai/chrome-data"
    },
    "discussion": {
      "end_signal": "<已得出最终结果>",

      "timeout_seconds": 120,
      "arbitrator": "auto"

        "enabled": false,
        "selectors": {

          "send_button": "button[data-testid='send-button']",
          "stop_button": "button[data-testid='stop-button']",
          "response_container": "div[data-testid='messages']",
          "last_response": "div[data-testid='messages'] > div:last-child",
          "login_indicator": "button[aria-label='User menu']",

        }
      }
    ]
  }
UI设置界面：Streamlit 侧边栏提供"⚙️ 设置"入口，展开后用户可：
添加 / 编辑 / 删除 AI 平台（名称、URL、各类选择器）
修改 LM Studio 配置（启用状态、URL、显示名称）
修改讨论参数（最大轮数、超时时间、拍板方）
保存时写入 config.json，并刷新运行时配置
启用与登录检测：用户在主界面点击某AI的"启用"按钮 →
ChromeManager 自动启动调试模式 Chrome（--remote-debugging-port=9222 --user-data-dir=... --remote-allow-origins=*）
Playwright 通过 connect_over_cdp 连接，打开该AI的URL
执行多方法组合登录检测：
检测是否存在"登录"按钮（若存在 → 未登录）
检测是否存在用户头像/用户菜单（若存在 → 已登录）
检测输入框是否可交互（若不可交互 → 可能未登录）
检测URL是否包含 login / auth（若包含 → 未登录）
若未登录 → UI 提示"请在弹出的Chrome窗口中手动登录"，并提供"✅ 确认已登录"按钮重新检测
登录成功 → 状态更新为"🟢 已就绪"
2. 托管模式（全自动讨论）
界面布局：
左侧：AI平台列表 + 启用/状态
中间：对话流展示区（st.chat_message 逐条展示每位AI的发言）
底部：话题输入框 + 文件上传（.txt / .md / .csv）+ "🚀 开始讨论"按钮
讨论流程：
用户选择2位已启用的AI
输入话题（若上传文件，utils.py 读取文件内容并拼接到话题前缀）
点击"开始讨论"
第一轮：向AI-A发送话题 + 规则模板（含对方名称），提取AI-A的回复
第二轮：将AI-A的回复发送给AI-B（附加规则提醒），提取AI-B的回复
循环：交替进行，每轮将上一位AI的回复发送给下一位
结束判定：
主策略：检测回复中是否包含哨兵标记 <已得出最终结果>
备选：达到 max_rounds 上限
超时：单轮等待超过 timeout_seconds
回复提取与等待：
发送消息后，先检测"停止生成"按钮是否出现（表示AI正在输出）
等待"停止生成"按钮消失（表示AI输出完成）
备选策略：每隔2秒抓取最新回复内容，若连续3次内容不变则认为输出完成
提取最新一条回复的纯文本（去除HTML标签、按钮文本等）
拍板机制（arbitrator）：
"auto"（默认）：谁先发出 <已得出最终结果>，谁就是拍板方。提取该AI标记前的所有内容作为最终方案。
"指定AI名称"（如 "ChatGPT"）：讨论结束后，将完整对话历史发送给指定AI，要求其汇总并输出最终方案。
"lm_studio"：讨论结束后，将完整对话历史通过 OpenAI 兼容接口发送给本地 LM Studio 模型，由其综合输出最终方案。
结果展示：讨论结束后，在对话流底部用 st.success() 高亮展示最终方案。
3. 圆桌模式（用户主导）
界面布局：
左侧：AI平台列表 + 参与者选择（勾选2-3位AI）
中间：st.chat_message 对话流（用户 + 多位AI的消息交错展示，每位AI有不同头像/颜色）
底部：st.chat_input 输入框 + 发送目标选择（发给某位AI / 发给所有AI）
交互流程：
用户勾选参与AI
用户输入消息并选择发送目标
系统自动将消息发送到目标AI的网页，等待回复，提取并展示
若选择"发给所有AI"，则并行发送并依次展示回复
用户可随时插话、追问、引导方向
拍板权完全在用户手中，用户可随时点击"📌 拍板"按钮，将当前讨论历史交给指定AI或LM Studio生成最终方案
本地AI辅助（可选）：若启用 LM Studio，在侧边栏展示：
📋 实时讨论摘要（每3轮更新一次）
❓ 追问建议（本地模型生成3个建议问题，用户点击即可发送）
这些辅助信息绝不干扰主对话流
4. 本地模型增强（LM Studio）
调用 LM Studio 提供的 OpenAI 兼容接口（默认 http://127.0.0.1:1234/v1）

模型不写死，由 LM Studio 当前加载的模型决定（client.models.list() 获取）
界面显示用户自定义名称（如 "MyAi"）

import json

class ChromeManager:
    def __init__(self, config):
        self.config = config
        self.chrome_process = None
        self.playwright = None
        self.browser = None
        each_time

    async def start_chrome_debug(self):
        """启动 Chrome 调试模式"""
        # 构造 Chrome 启动命令
        cmd = [
            "google-chrome",
            f"--remote-debugging-port={self.config.chrome.debug_port}",
            f"--user-data-dir={self.config.chrome.user_data_dir}",
            "--remote-allow-origins=*"
        ]
        # 启动进程
        self.chrome_process = subprocess.Popen(cmd, stdout=subprocess.PIPE, stderr=subprocess.PIPE)
        # 等待端口就绪 (最多等待10秒)
        await asyncio.sleep(2)  # 简化示例，实际应检测端口
        # 连接 Playwright
        await self.connect_playwright()

    async def connect_playwright(self):
        """连接已运行的 Chrome 调试实例"""
        self.playwright = await async_playwright().start()
        self.browser = await self.playwright.chromium.connect_over_cdp(
            f"http://127.0.0.1:{self.config.chrome.debug_port}"
        )
        return self.browser

    async def send_and_wait(self, page, message: str, ai_config: dict) -> str:
        """发送消息并等待回复，提取纯文本回复"""
        try:
            # 1. 定位输入框并输入
            input = await page.wait_for_selector(
                ai_config["selectors"]["input_textarea"],
                state="visible"
            )
            await input.fill(message)
            # 2. 点击发送按钮
            send_button = await page.wait_for_selector(
                ai_config["selectors"]["send_button"],
                state="visible"
            )
            await send_button.click()
            # 3. 等待回复 (主策略: 等待停止按钮消失)
            await page.wait_for_selector(
                ai_config["selectors"]["stop_button"],
                state="detached",  # 等待元素从DOM中移除
                timeout=self.config.discussion.timeout_seconds * 1000
            )
            # 4. 提取最新回复
            last_response = await page.query_selector(
                ai_config["selectors"]["last_response"]
            )
            return await last_response.inner_text()
        except Exception as e:
            raise Exception(f"发送消息失败: {e}")

class RoundTableMode:
    @staticmethod
    async def send_and_wait(prompt: str, target_ai: dict, config: dict) -> str:
        """圆桌模式：发送消息给指定AI并等待回复"""
        browser = await ChromeManager.connect_playwright()

            if target_ai["url"] in p.url:
